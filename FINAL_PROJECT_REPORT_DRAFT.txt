[SAMPLE SHEET 5: ABSTRACT]

ABSTRACT

PharmaSync is an advanced, localized inventory management solution designed to optimize the operations of independent pharmacy stores. The system addresses the critical challenges of medicinal stock management, including batch-wise tracking, expiry-sensitive waste reduction, and real-time tax-compliant billing. Built using a high-performance stack comprising Django (Backend), React 19 (Frontend), and PostgreSQL (Database), PharmaSync implements intelligent algorithms such as First-In-First-Out (FIFO) stock deduction to prioritize the sale of near-expiry medicines. Key features include a "Smart-Match" alternative medicine engine, a multi-level automated expiry notification system (Critical, Warning, Safe), and a dual-stream return management module for separate handling of customer and supplier returns. The system also provides data-driven analytics through dynamic visualizations, identifying fast-moving items and dead stock to prevent capital stagnation. PharmaSync ensures data integrity through JWT-based security and detailed audit logging of every inventory adjustment.

[CHAPTER 1: INTRODUCTION]

1.1 OVERVIEW
In the modern healthcare landscape, local pharmacies act as the first line of medical defense. However, many still rely on manual registers or basic software that fails to handle the sensitivity of medicine batches. PharmaSync is developed to modernize these operations, ensuring that stock is tracked with surgical precision to prevent patient risk and financial loss.

1.2 PROBLEM STATEMENT
Pharma stores face high operational friction due to:
- Expiry Wastage: No automated system to prioritize selling medicines near expiry.
- Manual Errors: Human mistakes in recording batch numbers and cost prices.
- Information Gap: Difficulty in finding chemical alternatives when a requested brand is unavailable.
- Audit Difficulty: Inefficient tracking of supplier returns for damaged or expired stock.

1.3 MOTIVATION
The primary motivation is to reduce medicinal waste and improve the efficiency of local pharmacists. By automating the stock lifecycle—from procurement to sale—we can ensure that patients receive fresh medicines while the pharmacy operates at peak financial efficiency.

[CHAPTER 2: SYSTEM ANALYSIS]

2.1 SOFTWARE REQUIREMENTS
- Backend: Django 5.0, Django REST Framework
- Frontend: React 19, Vite, TypeScript
- Database: PostgreSQL
- Authentication: JWT (SimpleJWT)
- Visualization: Recharts

2.2 HARDWARE REQUIREMENTS
- Processor: Dual Core 2.0GHz+
- RAM: 4GB minimum
- Camera: For potential OCR/Barcode integration

2.3 TECHNOLOGY STACK JUSTIFICATION
The choice of React 19 ensures a highly responsive and fluid single-page application (SPA). Django provides high-level security features and rapid development for the RESTful API. PostgreSQL ensures relational data integrity, which is vital for matching medicines with their respective batches and sales records.

[CHAPTER 3: PROPOSED APPROACH & ALGORITHMS]

3.1 FIFO STOCK HANDLING ALGORITHM
To minimize wastage, the system employs First-In-First-Out (FIFO) logic:
1. Input: Medicine ID and Quantity required.
2. Process: Fetch all batches for the ID -> Sort by Expiry Date (Ascending).
3. Deduction: Pick the earliest expiring batch -> Deduct quantity -> If batch empty, move to next oldest.
4. Output: Updated batch levels and consolidated line item for billing.

3.2 NEAR-EXPIRY DETECTION LOGIC
The system runs a background calculation:
- Days_Left = Expiry_Date - Today.
- If Days_Left <= 30: "Critical" (High priority alert).
- If Days_Left <= 90: "Warning" (Moderate priority).
- Action: Displayed on Dashboard and dedicated Expiry Report page.

---------------------------------------------------------------------------

[CHAPTER 4: CORE MODULES & IMPLEMENTATION]

4.1 INVENTORY & BATCH MANAGEMENT
Unlike traditional systems, PharmaSync separates the 'Medicine' profile from the 'Batch'. One medicine can have multiple batches with different prices and expiries.

4.2 BILLING & GST ENGINE
Real-time billing with dynamic search. As medicines are added to the cart, the system calculates GST per item and aggregates the total tax and grand total automatically.

4.3 SMART-MATCH ALTERNATIVE SUGGESTION
When a medicine is out of stock, the pharmacist can click the "Alternatives" button. The system queries the database for medicines with the same Generic Name or Category that are currently in stock.

[CHAPTER 5: CONCLUSION & FUTURE SCOPE]

5.1 CONCLUSION
PharmaSync provides a comprehensive solution for local pharmacy management. It successfully automates complex tasks like FIFO stock deduction and multi-level expiry alerts, which are often the biggest pain points for small-scale pharmacies.

5.2 FUTURE SCOPE
Future enhancements include:
- Integrated Barcode/QR Scanning for instant stock-in/billing.
- Automated Email Notifications to suppliers for near-expiry returns.
- Mobile Application for remote inventory monitoring.
